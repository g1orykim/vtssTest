Trunk Configuration Help

This page is used to configure Hash Code Contributors and Trunk Membership.

The Hash Code Contributors settings are global, whereas the Trunk Membership relates to the currently selected stack unit, as reflected by the page header.

Hash Code Contributors

Source MAC Address
The Source MAC Address can be used to calculate the destination port for the frame. Check to enable the use of the Source MAC Address, or uncheck to disable. By default, Source MAC Address is enabled.
Destination MAC Address
The Destination MAC Address can be used to calculate the destination port for the frame. Check to enable the use of the Destination MAC Address, or uncheck to disable. By default, Destination MAC Address is disabled.
IP Address
The IP address can be used to calculate the destination port for the frame. Check to enable the use of the IP Address, or uncheck to disable. By default, IP Address is enabled.
TCP/UDP Port Number
The TCP/UDP port number can be used to calculate the destination port for the frame. Check to enable the use of the TCP/UDP Port Number, or uncheck to disable. By default, TCP/UDP Port Number is enabled.

Trunk Membership

Trunk
Indicates the trunk ID for the settings contained in the same row. Trunk ID "Normal" indicates port that are not part of any trunk.
Locality
Indicates the trunk type. This field is only valid for stackable switches.
Global: The trunk members may reside on different units in the stack. The device supports two 8-port global trunks.
Local: The trunk members reside on the same unit. Each local trunk may consist of up to 16 members.
Name
An optional trunk name. Valid characters are a-z, A-Z, 0-9, space, underscore and dash.
Mode
Indicates the trunk mode.
Disabled: Trunk is disabled, no traffic will flow and LACPDUs will be dropped. But the ports that are members of the trunk will not be released.
Static: Select to enable a trunk to operate in static mode. A static trunk does not require a partner system to be able to aggregate its member ports. In this mode, it does not transmit or process received LACPDUs i.e. the member ports do not transmit LACPDUs and all the LACPDUs they may receive are dropped.
LACP Active: Trunk will be initiated and maintained by the periodic exchanges of LACPDUs.
LACP Passive: Trunk will only participate if the other end sends LACPDUs (other end is active).
Port Members
Each switch port is listed for each trunk. Select a radio button to change the trunk membership for a specific port. By default, no ports belongs to any trunk. Only full duplex ports can join a trunk and ports must be in the same speed in each trunk.

Buttons
